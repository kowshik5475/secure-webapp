Security Controls Implemented

🔑
A02

bcrypt Password Hashing

Passwords are hashed with bcrypt (work factor 12) before storage. Plaintext passwords never touch the database. bcrypt is adaptive — work factor can be increased as hardware improves.

bcrypt.hashpw(pw, bcrypt.gensalt(rounds=12))
🍪
A05

JWT in httpOnly Cookie

Session tokens are stored in httpOnly, Secure, SameSite=Strict cookies — not localStorage. JavaScript cannot read the token, eliminating XSS-based session theft.

httponly=True, samesite="Strict"
✅
CSRF

Synchronizer Token Pattern

Every state-changing form embeds a CSRF token from the signed JWT. The server validates it on every POST/PUT/DELETE. SameSite=Strict adds a second defence layer.

<input type="hidden" name="_csrf_token" value="...">
👤
A01

Role-Based Access Control

Admin vs. User roles are enforced server-side on every protected route via decorators. Client-side role checks are UX only. Role changes are logged to the audit trail.

@admin_required — server-side only
🛡
A03

Parameterized SQL

Every database query uses ? placeholders — no string concatenation. SQLite's parameterized API treats values as data, never as executable SQL. SQL injection is structurally impossible.

conn.execute("SELECT * FROM users WHERE email = ?", (email,))
🖥
XSS

Output Encoding

Jinja2 auto-escaping is enabled for all .html templates. Every {{ variable }} is HTML-escaped before output. The | safe filter is never used on user-controlled data.

{{ user_input }} → auto-escaped
📎
A04

File Upload Validation

File type is validated by magic bytes (not extension). Files are stored outside the web root with randomised UUID names. Path traversal and extension spoofing are both mitigated.

filetype.guess(bytes) → checks magic bytes
⏱
A07

Rate Limiting + Lockout

Login is rate-limited to 10 requests/minute via Flask-Limiter. After 5 consecutive failures, the account is locked for 15 minutes. Thwarts brute-force and credential-stuffing attacks.

@limiter.limit("10 per minute")
📋
A09

Audit Logging

Every security event (login, failure, lockout, role change, file upload, admin action) is written to the audit_log table with timestamp, actor, target, IP, and details.

write_audit_log(event_type, user_id, ip, ...)
🛡️
A05

Secure HTTP Headers

Every response carries CSP, X-Content-Type-Options, X-Frame-Options, Referrer-Policy, Permissions-Policy, and HSTS headers. Applied globally via Flask after_request().

Content-Security-Policy: default-src 'self'
